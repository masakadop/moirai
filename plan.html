🎭 PNGTuber Web — 開発プラン

スマホのブラウザとカメラだけで完結する PNGTuber 表示ページを GitHub Pages で公開する

1. コンセプト

インストール不要・サーバ不要で、スマホ 1 台だけで PNGTuber(立ち絵の口パク・まばたきアバター)を動かせる Web ページを作る。 ブラウザの getUserMedia でフロントカメラの映像を取得し、顔認識(MediaPipe Face Landmarker)で 口の開閉・まばたき・顔の傾きを検出して、PNG 立ち絵をリアルタイムに切り替え・アニメーションさせる。

  • 静的GitHub Pages のみで動作(ビルド不要、サーバサイド処理なし)
  • プライバシーカメラ映像は端末内で処理し、外部送信しない
  • モバイル完結画像の読み込みから配信画面の表示まで、すべてスマホで操作可能

2. 主要ユースケース

  1. スマホでページを開く → カメラ許可 → 自分の立ち絵 PNG を選択 → 喋ると口パクするアバターが全画面表示される
  2. 雑談配信・通話でスマホ画面をそのまま見せる、または PC の OBS のブラウザソースとして同じページを読み込む
  3. 立ち絵を持っていない人向けに、サンプルアバターを同梱してすぐ試せるようにする

3. 機能一覧

MVP(必須)

機能内容
カメラ入力getUserMedia でフロントカメラを取得。カメラ切替(イン/アウト)対応
口パク検出MediaPipe Face Landmarker の blendshape(jawOpen など)で口の開閉を判定し、口閉じ/口開き画像を切替
まばたき検出eyeBlinkLeft / eyeBlinkRight で目閉じ画像に切替
画像アップロード口閉じ・口開き・目閉じの PNG を <input type="file"> で読み込み(端末内処理のみ)
アバター描画Canvas に描画。喋っているあいだは軽く揺れる(バウンス)演出
背景色変更単色背景(グリーンバック含む)をカラーピッカーで変更 → OBS のクロマキーに対応
設定の保存感度・背景色・画像を localStorage / IndexedDB に保存し、次回起動時に復元
サンプルアバター同梱のサンプル画像で、画像を用意しなくても即体験できる

拡張(後続フェーズ)

  • 顔の傾き(roll)に追従してアバターを回転
  • マイク音量ベースの口パク(顔認識が使えない環境向けフォールバック)
  • 表情プリセット(喜び・驚きなど)のタップ切替、複数コスチューム
  • 感度・しきい値の調整 UI(スライダー)
  • PWA 化(ホーム画面追加・オフライン起動)

5. 画面構成

  • メイン画面(index.html): アバター全画面表示。タップで設定パネルの表示/非表示をトグル
  • 設定パネル(オーバーレイ): 画像選択(口閉じ/口開き/目閉じ)、背景色、感度スライダー、カメラ切替、カメラプレビュー(小窓・オフ可)
  • ステータス表示: カメラ許可待ち / モデル読込中 / 顔未検出 などの状態を明示

スマホ縦持ちを基準にデザインし、横持ち・PC 表示(OBS ブラウザソース)にもレスポンシブ対応する。

7. 開発フェーズ

Phase 1 — MVP(まずここまで) カメラ取得 → 顔認識 → サンプルアバターの口パク・まばたきが動く index.html を GitHub Pages で公開する。
Phase 2 — カスタマイズ 画像アップロード、背景色変更、感度調整、設定の永続化。
Phase 3 — 品質向上 顔の傾き追従、マイク音量フォールバック、表情プリセット、PWA 化。

8. リスクと対策

リスク対策
iOS Safari のカメラ制約(ユーザー操作起点でないと再生不可 など)「開始」ボタンのタップを起点にカメラとモデルを初期化する
MediaPipe の WASM/モデル読込が遅い・失敗する読込中インジケータ表示。失敗時はマイク音量口パクへフォールバック(Phase 3)
スマホでの発熱・電池消費推論フレームレートを制御(例: 24fps 上限)。カメラプレビューはオフ可能に
バックグラウンドタブで停止するWake Lock API + 画面常時点灯の案内を表示
プライバシー懸念映像を外部送信しないことを README とページ内に明記

9. 実装用プロンプト(コピーして使えます)

次のステップで実装を依頼するときは、以下のプロンプトをそのまま使ってください。

GitHub Pages で公開できる「ブラウザ完結型 PNGTuber」を実装してください。

## ゴール
スマホのブラウザだけで、フロントカメラの顔認識により PNG 立ち絵が
口パク・まばたきするページを作る。サーバ処理・ビルドツールは使わず、
静的ファイルのみで構成する。

## 要件
1. index.html をエントリポイントとし、素の HTML/CSS/JS(ES Modules)で実装する
2. getUserMedia でフロントカメラを取得する(「開始」ボタンのタップを起点に初期化)
3. MediaPipe Tasks Vision の Face Landmarker(CDN 読込)で blendshape を取得し、
   - jawOpen で口の開閉 → 口閉じ/口開き画像を切替
   - eyeBlinkLeft/Right で目閉じ画像に切替
   - 発話中はアバターを軽く揺らす
4. 立ち絵は「口閉じ」「口開き」「目閉じ」の 3 枚を file input で読み込めるようにし、
   IndexedDB に保存して次回復元する。未設定時は同梱のサンプル画像を使う
5. 背景は単色(カラーピッカーで変更、デフォルトはグリーンバック #00FF00)
6. 検出感度(口・目のしきい値)をスライダーで調整でき、localStorage に保存する
7. 画面はスマホ縦持ち基準のレスポンシブ。アバターは全画面表示し、
   画面タップで設定パネルをトグル表示する
8. カメラ映像は端末内でのみ処理し、外部送信しないことをページ内に明記する
9. カメラ許可待ち・モデル読込中・顔未検出のステータスを表示する
10. GitHub Pages(リポジトリ直下配信)でそのまま動くパス構成にする

## 成果物
- index.html / css / js / サンプルアバター画像
- README.md(使い方・GitHub Pages の有効化手順・OBS ブラウザソースでの取込方法)
